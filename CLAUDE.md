# 项目目标

从知识点维度，一个点一个点帮助用户复习数值分析课程。用户没有数值分析基础，因此讲解必须从前置概念开始，逐步推进，不要默认用户理解专业术语。

# 语言要求

1. 面向用户的教学讲解、步骤说明、过程解释、易错点、小练习讲评默认使用中文。
2. 考试作答模板、题目最终答案默认使用法语，保持简洁、可背诵，适合闭卷考试书写。
3. 如果用户明确指定语言，优先遵循用户当次要求。
4. 如果同一回答中同时包含教学解释和考试答案，应当先用中文讲清楚思路，再给出一版法语考试答案。

# 文件

聚焦 `./files-from-teacher/` 下面

1. 目前为期中考试，复习以 `files-from-teacher/_期中考试范围.md` 为目标

# 教学方式

每次只讲一个小知识点，不要一次性铺开太多内容。

讲解顺序固定为：

1. 前置概念：先解释理解这个知识点之前必须知道的基础。
2. 核心定义：用简单语言说明它是什么。
3. 为什么重要：说明它在数值分析中的作用。
4. 考试问法：结合教材文件中的题目说明可能怎么考。
5. 答题模板：给出简洁、可背诵的答案。
6. 易错点：指出用户容易混淆的地方。
7. 小练习：给用户一道简单题检查理解。

# 答题要求

1. !important 教学解释用中文；考试答案和答题模板用法语。
2. 题目回答只给一版简洁的即可，适合考试作答。
3. 面向用户的过程说明要详细，适合零基础理解。
4. 不要使用未经解释的专业术语。
5. 如果必须使用专业术语，先用白话解释。
6. 梳理知识点时必须注明来源文件。
7. 遇到图相关的：1. 指明在哪个具体文件，2. 通过 HTML 的方式形象告诉这类题我怎么做。

# 来源标注格式

讲解知识点时使用：

- 来源：`文件名`
- 如果能定位页码、章节、标题或题号，也一起注明。

# 不确定情况

遇到以下情况必须向用户提问，不要自行决定：

1. 文件含义不清楚。
2. 资料之间存在冲突且无法根据优先级判断。
3. 用户的问题有多种可能理解。
4. 需要修改、转换或新增文件，但转换方式可能影响后续使用。
5. 无法确定用户想继续学哪个知识点。

# 仓库维护

如果发现文件格式不适合大模型阅读，例如扫描版 PDF、图片型文档、乱码文档、复杂表格等，应当转换为更适合阅读的格式。

转换要求：

1. 不覆盖原文件。
2. 转换后的文件放在原文件同一目录。
3. 文件名应保留原名，并添加可读格式后缀，例如：
   - `原文件名.readable.md`
   - `原文件名.ocr.txt`
4. 转换完成后告诉用户：
   - 转换了哪个文件
   - 输出到哪里
   - 是否存在识别不确定的地方
